Active Directory Home Lab
Windows Server 2025 · VirtualBox · blaborte.local
Completed

Built a fully functional Active Directory environment from scratch on a personal homelab using VirtualBox. Simulated a real corporate IT environment complete with domain controllers, organizational units, group policies, and user lifecycle management — replicating the infrastructure found in enterprise environments and directly applicable to IT admin and helpdesk roles.

IT Support Windows Server 2025 AD DS GPO PowerShell DNS / DHCP VirtualBox Splunk LDAP
Objectives
  • Simulate a corporate Windows domain environment for hands-on IT administration practice
  • Learn AD DS architecture — forests, domains, trees, sites, and trust relationships
  • Practice Group Policy Object (GPO) design, inheritance, and enforcement across OUs
  • Understand user provisioning, role-based access control, and OU delegation
  • Integrate DNS, DHCP, and LDAP services within the domain infrastructure
  • Forward security event logs to Splunk for SIEM correlation and threat detection practice
What Was Built
  • Deployed Windows Server 2025 as a Primary Domain Controller (PDC) with AD DS, DNS, and DHCP roles installed and configured
  • Created a second Server 2025 VM as a Backup Domain Controller for redundancy and failover testing
  • Structured OUs mirroring a real org: Departments → IT, HR, Finance, Management with nested sub-OUs per team
  • Provisioned 50+ test user accounts via PowerShell bulk script simulating onboarding at scale
  • Configured 8 Group Policy Objects covering: password policy, account lockout, desktop lockdown, drive mapping, software restriction, and screensaver timeout
  • Joined Windows 10 and 11 client VMs to the domain and verified login, policy application, and access control
  • Set up a Read-Only Domain Controller (RODC) to simulate branch office / remote site scenarios
  • Forwarded Security event logs (Event ID 4624, 4625, 4740) to Splunk via Universal Forwarder for SIEM correlation
PowerShell — Bulk User Provisioning
# Bulk create 50 users across departments $departments = @("IT","HR","Finance","Management") $password = ConvertTo-SecureString "Lab@2024!" -AsPlainText -Force foreach ($dept in $departments) { 1..12 | ForEach-Object { New-ADUser -Name "$dept.User$_" ` -GivenName "$dept" -Surname "User$_" ` -UserPrincipalName "$dept.user$_@blaborte.local" ` -Path "OU=$dept,DC=blaborte,DC=local" ` -AccountPassword $password -Enabled $true } }
Step-by-Step Setup Guide
01
Create VMs in VirtualBox
In VirtualBox Manager, create two VMs for domain controllers and one or two Windows 10/11 client VMs. Assign each at least 2 vCPUs and 4GB RAM. Set the network adapter to Internal Network (same network name, e.g. "adlab") so all VMs can communicate with each other, and add a second NAT adapter on the DC for internet access.
Download the Windows Server 2025 ISO from Microsoft's Evaluation Center (180-day free eval). In VirtualBox, attach it under VM Settings → Storage → Controller: IDE → add optical drive.
02
Install Windows Server 2025 & Set a Static IP
Install Windows Server 2025 (Desktop Experience) on your primary VM. Once booted, set a static IP before promoting to a domain controller — DNS must be stable.
# In PowerShell (on the Server VM) New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.10.10 -PrefixLength 24 -DefaultGateway 192.168.10.1 Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses 127.0.0.1
03
Install AD DS & DNS Roles
Open Server Manager → Add Roles and Features → select Active Directory Domain Services and DNS Server. After installation, click the flag notification to promote the server to a Domain Controller.
# Or via PowerShell Install-WindowsFeature -Name AD-Domain-Services, DNS -IncludeManagementTools
04
Promote to Domain Controller & Create the Forest
Run the AD DS Configuration Wizard. Choose "Add a new forest" and set your domain name. Set the Forest and Domain Functional Levels to Windows Server 2016 or higher. Create a DSRM password, then let the server restart.
# Via PowerShell Install-ADDSForest -DomainName "blaborte.local" -DomainNetbiosName "BLABORTE" -InstallDns -Force
After reboot, log in with BLABORTE\Administrator. The server is now a Domain Controller.
05
Create Organizational Units (OUs)
Open Active Directory Users and Computers (ADUC). Right-click your domain → New → Organizational Unit. Create top-level OUs for each department, then nest sub-OUs as needed.
# Via PowerShell $base = "DC=blaborte,DC=local" foreach ($ou in @("IT","HR","Finance","Management")) { New-ADOrganizationalUnit -Name $ou -Path $base }
06
Bulk Create User Accounts via PowerShell
Rather than creating users one-by-one in ADUC, use PowerShell to provision users at scale. This simulates real onboarding workflows and is great for GPO testing.
$password = ConvertTo-SecureString "Lab@2024!" -AsPlainText -Force $departments = @("IT","HR","Finance","Management") foreach ($dept in $departments) { 1..12 | ForEach-Object { New-ADUser -Name "$dept.User$_" ` -UserPrincipalName "$dept.user$_@blaborte.local" ` -Path "OU=$dept,DC=blaborte,DC=local" ` -AccountPassword $password -Enabled $true } }
07
Configure Group Policy Objects (GPOs)
Open Group Policy Management (gpmc.msc). Right-click an OU → Create a GPO and Link it. Edit the GPO to configure settings. Apply and test by running gpupdate /force on a client.
# Force policy refresh on client gpupdate /force # Check applied policies gpresult /r
Start with Password Policy (under Computer Config → Windows Settings → Security Settings → Account Policies). This applies domain-wide from the Default Domain Policy.
08
Join Client VMs to the Domain
On each Windows 10/11 client VM, set the DNS server to point to your DC's IP. Then join the domain via Settings → System → About → Rename this PC (Advanced).
# Set DNS to point to DC Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses 192.168.10.10 # Join domain via PowerShell Add-Computer -DomainName "blaborte.local" -Credential (Get-Credential) -Restart
09
Set Up a Backup Domain Controller
Install Windows Server 2025 on a second VM, set a static IP, and point its DNS to the primary DC. Then join it to the domain and promote it as an additional domain controller.
# On the second server, after joining the domain: Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools Install-ADDSDomainController -DomainName "blaborte.local" -InstallDns -Credential (Get-Credential) -Force
10
Forward Event Logs to Splunk
Install the Splunk Universal Forwarder on your Domain Controller. Configure it to forward Windows Security event logs to your Splunk instance for SIEM monitoring and alerting.
# inputs.conf on the forwarder [WinEventLog://Security] disabled = 0 index = wineventlog evt_resolve_ad_obj = 1
Key events to watch: 4625 (failed logon), 4740 (lockout), 4728 (group membership change). Build Splunk dashboards and alerts around these for SOC practice.
